## Документація по розробленому веб-застосунку

### Призначення застосунку
**Simple Inventory** - це REST API сервіс обліку обладнання. Об'єкт інвентарю містить наступні поля: `id`, `name`, `quantity`, `created_at`. Застосунок дозволяє зберігати записи про інвентар, переглядати загальний список предметів та отримувати детальну інформацію по кожному з них.

### Середовище розробки та запуску
* **Мова та фреймворк:** Python 3, Flask.
* **База даних:** MariaDB (через PyMySQL).
* **Оточення:** Запуск виконується у віртуальному середовищі Python (`venv`) на ОС CentOS Stream 9. У робочому середовищі застосунок запускається за допомогою systemd-unit від окремого системного користувача `app`. Перед запуском автоматично виконується скрипт міграції бази даних.

### Документація по АРІ-ендпоінтах
Застосунок віддає різні результати залежно від заголовку `Accept` (html-сторінку або application/json).

* **Ендпоінти для перевірки стану:**
  * `GET /health/alive` - завжди повертає НТТР 200 з вмістом ОК.
  * `GET /health/ready` - повертає НТТР 200 з вмістом ОК якщо сервіс успішно підключився до БД і готовий обробляти запити, інакше повертає НТТР 500.
* **Бізнес-логіка:**
  * `GET /` - Кореневий ендпоінт, який очікує і віддає тільки `text/html` і повертає список усіх ендпоінтів бізнес-логіки застосунку.
  * `GET /items` - вивести список усіх предметів в інвентарі (id, name).
  * `POST /items` - створити новий запис у системі обліку (очікує параметри `name`, `quantity`).
  * `GET /items/<id>` - вивести детальну інформацію по запису в інвентарі (id, name, quantity, created_at).

## Документація по розгортанню

### Вимоги до ресурсів віртуальної машини
Усі компоненти розгортаються на одній віртуальній машині Linux.
* **Базовий образ:** [CentOS Stream 9 (x86_64)](https://www.centos.org/centos-stream/) - `minimal install`.
* **CPU:** Мінімум 2 ядра.
* **RAM:** 2048 MB.
* **Disk:** 20 GB.

### Налаштування при встановленні OS
1. В інсталяторі у розділі "Network & Host Name" необхідно увімкнути мережевий адаптер (положення **ON**).
2. Під час створення користувача необхідно створити вашого користувача для роботи з проєктом - `student` і надати йому адміністративні права, або можливість ескалації до root (поставити відповідну галочку).

### Як увійти на ВМ
* **Credentials за замовчуванням:** Логін: `student` / Пароль: 1234.
* *Примітка:* Скрипт автоматизації блокує "дефолтного" користувача в системі, щоб після первинної конфігурації можна було увійти тільки з-під створених користувачів (`student`, `teacher`, `operator`).

## Інструкція з тестування розгорнутої системи

Для перевірки коректності роботи системи виконайте наступні команди з термінала:

**1. Перевірка reverse proxy (Nginx):**
Клієнти взаємодіють із системою через reverse proxy (nginx).

Отримати головну сторінку зі списком ендпоінтів:
```bash
curl http://localhost/
```
Створити новий предмет інвентаря:
```bash
curl -X POST http://localhost/items -H "Content-Type: application/json" -d '{"name": "Server Rack", "quantity": 2}'
```
Переглянути список усіх предметів:
```bash
curl -H "Accept: application/json" http://localhost/items
```

**2. Перевірка безпеки та обмежень:**
Доступ до бази даних має бути обмежений лише з віртуальної машини, а назовні nginx повинен віддавати лише кореневий ендпоінт та ендпоінти бізнес-логіки застосунку.
```bash
curl -I http://localhost/health/alive
```
*Очікуваний результат:* Сервер Nginx повинен повернути помилку 403 Forbidden.

**3. Перевірка прав доступу користувача operator:**
Користувач operator повинен мати право виконувати тільки такі команди: запуск веб-застосунку, зупинку веб-застосунку, перезапуск веб-застосунку, перегляд статусу веб-застосунку, перезавантаження конфігурації (reload) nginx.
```bash
su - operator
sudo systemctl restart mywebapp
sudo systemctl status mywebapp
```
*Очікуваний результат:* Команди виконуються успішно. Будь-які інші команди через sudo (наприклад, sudo dnf update) будуть відхилені системою.

## Завантаження та запуск автоматизації розгортання
Розгортання віртуальної машини повністю автоматизовано за допомогою bash-скрипта setup.sh. Автоматизація при запуску виконує всі необхідні кроки (встановлення пакетів, створення користувачів, створення БД, налаштування сервісів та Nginx).

### Як запустити:
1. Завантажте файли `app.py`, `migrate.py` та `setup.sh` на віртуальну машину.
2. Надайте скрипту права на виконання:
```bash
chmod +x setup.sh
```
3. Запустіть скрипт з правами адміністратора:
```bash
sudo ./setup.sh
```
